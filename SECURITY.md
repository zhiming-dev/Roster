# Security Policy

## Reporting a vulnerability

Do not open public GitHub issues for suspected security vulnerabilities.

Report them privately to the maintainer with:

- a clear description of the issue,
- reproduction steps or a proof of concept,
- affected files or components,
- impact assessment, and
- any suggested remediation.

Until a dedicated security contact is published, coordinate through a private maintainer channel and avoid disclosing exploit details publicly before a fix is available.

## Scope

Please report issues involving:

- secret handling,
- approval gate bypasses,
- unintended destructive actions,
- unsafe runtime execution paths,
- authorization or capability boundary failures.
